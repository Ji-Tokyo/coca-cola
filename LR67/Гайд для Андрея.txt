Гайд о том, что я сделал.

Цель заключалась в создании CRUD. C - CREATE, R - READ, U - UPDATE, D - DELETE. То есть, создать интерфейс для твоей изначальной таблице

Использовался шаблон проектирования Table Module(по тз). В кратце расскажу что это, потому что может спросить.

- У тебя есть папка .core (В ней находятся все файлы cruda.) Щас пробежимся по ее содержимому
- index.php (В ней находятся все подключения .core файлов. Это для того, чтобы для любой страницы полключать только index.php и
    вместе с ним будут подтягиваться все файлы из .core)
- db.php (новый правильный файл подключения к базе данных. Использовался шаблон Singleton(единственный экземпляр класса), PDO)
- ObjectTalbe.php (файл в котором проводятся все операции с нашей базой данных)
- ObjectActions.php(подгототавливает данные для отдачи их ObjectTalbe.php)
- ObjectLogic.php(шаблонные проверки данных по типу является ли число числом, является ли строка строкой и тд...)

Вот эта вся папка по сути и есть CRUD. Его основа.

Твоя табличка находится в equipment.php. Любое действие(create,edit,delete) это модульное окно с формой.
Форма отправляется на php_code/crud_actions.php, где я вызываю ObjectActions.php, которое выполняет вызываемое действие.
Затем перенаправляю обратно на equipment.php

Почему не сразу вызываю ObjectActions.php в equipment.php?
Потому что нам нужно обновление страницы после изменений данных в БД.
Чтобы отрисовалась актуальная информация в табличке.

Вроде это все изменения в этом проекте. Другое ничего не добавлял.
